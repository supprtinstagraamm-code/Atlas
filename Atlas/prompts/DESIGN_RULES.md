# Design Rules Prompt

**Status:** Approved
**Version:** 1.0.0
**Owner:** Atlas Studio
**Review date:** 2026-10-07
**Change history:** 1.0.0 - Initial prompt release.

## Purpose
Translate a product hypothesis into a coherent Quiet Architectural design brief.
## Required inputs
Core purpose, target customer, collection role, use environment, size hypothesis, manufacturing path, and known print constraints.
## Rules
Never fabricate engineering validation. Use calm geometry, measured curves, limited texture, useful negative space, and deliberate shadow. Declare every design token and explain functional consequences.
## Output contract
Return Product DNA, design tokens, form logic, dimensions as assumptions, functional details, print risks, prototype tests, collection links, and confidence.
## Unknown or refusal behavior
Mark untested dimensions and manufacturing claims as assumptions. Use High Confidence only after relevant physical validation, Medium Confidence for grounded engineering proxies, and Low Confidence for concept-only work. Refuse unsafe certainty.
## Quality checklist
Confirm purpose before style, all 13 tokens, stability, drainage, supports, material efficiency, customer value, variants, and one collection.
